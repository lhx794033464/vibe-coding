看到 draw.io 一直停留在“Loading...”转圈界面，通常是因为**嵌入模式下的通信握手未完成**。draw.io 在嵌入模式下会等待父页面（你的网页）发送一条 `init` 或 `ready` 消息的响应，如果一直没有收到，就会一直显示加载动画。

下面我帮你逐步排查和解决。

---

## 🧭 第一步：检查 iframe 的 src 是否正确

确保你的 iframe 地址包含必需的参数：

```html
<iframe 
  id="drawioFrame" 
  src="https://embed.diagrams.net/?embed=1&proto=json" 
  style="width:100%; height:600px;">
</iframe>
```

- **必须使用 `https://embed.diagrams.net/`**，不要用其他域名。
- **必须带上 `?embed=1&proto=json`**，否则不会启用嵌入通信模式。

---

## 🔍 第二步：打开浏览器控制台查看错误信息

在页面按 `F12` 打开开发者工具，切换到 **Console** 面板，看看有没有红色的报错信息。常见错误有：

- `Blocked a frame with origin ...`：跨域或沙箱限制。
- `Failed to load resource: net::ERR_CONNECTION_REFUSED`：网络问题，可能无法访问 draw.io 服务器。
- 没有 `init` 消息相关的日志：说明父页面没有正确监听并回复。

如果有错误，请把具体信息告诉我，我可以帮你进一步分析。

---

## 🧪 第三步：确认父页面是否正确处理了消息通信

draw.io 在嵌入模式下会向父页面发送一条 `init` 消息，父页面需要接收并**在收到 `init` 后立即发送 `load` 或至少不报错**，否则 draw.io 会一直等待。

**最简单的正确示例**（仅保证通信，不加载任何图表）：

```html
<iframe id="drawioFrame" src="https://embed.diagrams.net/?embed=1&proto=json" style="width:100%; height:600px;"></iframe>

<script>
  const iframe = document.getElementById('drawioFrame');

  window.addEventListener('message', function(event) {
    // 重要：验证消息来源
    if (event.origin !== 'https://embed.diagrams.net') return;

    const data = event.data;
    console.log('收到 draw.io 消息:', data);

    // 当 draw.io 准备好时，它会发送 'init' 消息
    if (data === 'init') {
      console.log('draw.io 已就绪，现在可以发送数据');
      // 即使没有数据，也发送一个空 load 来让 draw.io 完成初始化
      iframe.contentWindow.postMessage({
        action: 'load',
        xml: '',   // 空 XML，draw.io 会显示空白画布
        autosave: 1
      }, 'https://embed.diagrams.net');
    }
  });
</script>
```

如果你没有添加这段 JavaScript，draw.io 就会一直转圈等待。**即使没有需要加载的图表，也必须发送 `load` 消息**。

---

## 🚧 第四步：检查网络是否正常

在 **Network** 面板中刷新页面，看 draw.io 的资源是否加载成功。如果大量请求显示红色（失败），可能是网络问题（例如国内访问 `embed.diagrams.net` 较慢或被阻断）。

**解决办法**：
- 使用代理或更换网络环境。
- 如果是在国内网络，可以尝试使用 draw.io 的国内镜像（如果有），但官方建议还是用 `embed.diagrams.net`。

---

## 🔒 第五步：检查 iframe 的 sandbox 属性

如果你的 iframe 加了 `sandbox` 属性，可能需要放开必要的权限：

```html
<iframe 
  src="https://embed.diagrams.net/?embed=1&proto=json"
  sandbox="allow-same-origin allow-scripts allow-popups allow-forms allow-modals"
  style="width:100%; height:600px;">
</iframe>
```

`allow-same-origin` 和 `allow-scripts` 是必须的，否则 draw.io 的脚本无法运行，也无法与父页面通信。

---

## 🧪 第六步：测试最简单的嵌入页面

创建一个全新的、最简单的 HTML 文件，只包含上面的 iframe 和消息监听代码，在本地或在线服务上运行。如果这个最小化示例能正常加载并显示空白画布，说明你的原页面有其他代码干扰（如重复监听、变量冲突等）。

---

## 🛠️ 第七步：使用官方提供的示例代码

draw.io 官方提供了一个最小的嵌入示例，你可以直接复制测试：

```html
<!DOCTYPE html>
<html>
<head>
    <title>draw.io 嵌入测试</title>
</head>
<body>
    <iframe id="drawioFrame" src="https://embed.diagrams.net/?embed=1&proto=json" style="width:100%; height:600px;"></iframe>

    <script>
        const iframe = document.getElementById('drawioFrame');

        window.addEventListener('message', function(event) {
            if (event.origin !== 'https://embed.diagrams.net') return;
            const data = event.data;
            if (data === 'init') {
                iframe.contentWindow.postMessage({
                    action: 'load',
                    xml: '',
                    autosave: 1
                }, 'https://embed.diagrams.net');
            }
        });
    </script>
</body>
</html>
```

将此文件保存为 `.html` 并打开，如果显示空白画布（没有转圈圈），说明嵌入成功。

# 金蝶产品智能问答

通过金蝶云社区智能服务接口，为用户解答金蝶产品使用过程中的各类问题。使用 PAT Token 认证调用 `/aisapi/ai-search` 接口，支持流式输出思考过程和回答。

**本技能无外部依赖**，token 存取能力已内置于脚本中。

## 使用场景

当识别到用户正在询问金蝶ERP产品的应用问题时，应调用此技能。以下是典型的触发场景和用户问法示例：

## 会话状态管理

技能需要维护以下会话状态：

| 状态项 | 说明 | 初始值 |
|-------|------|--------|
| `selected_product_id` | 用户选择的产品ID | null |
| `selected_product_name` | 用户选择的产品名称 | null |
| `session_id` | 多轮对话会话ID | null |

## 工作流程

当用户询问金蝶产品相关问题时，按以下步骤执行：

### 步骤1：初始化（一次性获取 Token 状态 + 产品列表）

**首次进入问答流程时，执行初始化命令：**

```bash
python <skill_dir>/scripts/cosmic_qa.py --init
```

**返回结果示例（Token 有效）：**

```json
{
  "type": "init",
  "token": { "valid": true, "token_preview": "kdt_xxxx...xxxx", "token_length": 64, "file": "C:\\Users\\xxx\\.kdclub\\pat_token.json" },
  "products": [
    {"productId": 3, "name": "金蝶AI星瀚"},
    {"productId": 93, "name": "金蝶AI套件"},
    ...
  ]
}
```

**返回结果示例（Token 无效）：**

```json
{
  "type": "init",
  "token": { "valid": false, "error": "未找到有效的 PAT Token..." },
  "products": [ ... ]
}
```

**处理逻辑：**

- **`token.valid: true`** → Token 已就绪，直接进入步骤2（产品选择），产品列表已在 `products` 字段中
- **`token.valid: false`** → 需要用户提供 Token，执行以下流程：

**引导用户获取 Token 的话术：**

> 使用金蝶产品智能问答需要先配置身份认证。请按以下步骤获取 PAT Token：
>
> 1. 打开浏览器访问 https://vip.kingdee.com
> 2. 登录您的金蝶云社区账号
> 3. 点击右上角头像 → 个人主页 → 编辑资料
> 4. 找到「个人访问令牌」区域 → 新建令牌
> 5. 复制生成的 token（格式如 `kdt_xxxxxxxx...`）
> 6. 将 token 发给我，我会自动保存供后续使用
>
> Token 只需配置一次，后续会话会自动读取。

**用户提供 Token 后，立即保存：**

```bash
python <skill_dir>/scripts/cosmic_qa.py --save-token "用户提供的token字符串"
```

**返回结果：**
```json
{"type": "token_saved", "message": "Token 已保存到 C:\\Users\\xxx\\.kdclub\\pat_token.json，后续会话无需重复提供。", "file": "C:\\Users\\xxx\\.kdclub\\pat_token.json"}
```

保存成功后告知用户"Token 已保存成功，后续无需重复提供"，然后使用 `--init` 返回中已有的 `products` 列表进入步骤2。

### 步骤2：产品选择（检查会话状态）

**检查会话中是否已有产品选择：**

- **如果 `selected_product_id` 已设置**：直接使用该产品，跳过选择步骤
- **如果 `selected_product_id` 为 null**：使用步骤1 `--init` 返回中的 `products` 列表展示给用户选择

### ⚠️ 极其重要：必须展示全部产品，严禁筛选或省略

**必须严格遵守以下铁律：**

1. **必须展示 `--list-products` 返回的全部产品**：返回多少个就展示多少个，一个都不能少
2. **严禁根据用户问题内容自行筛选产品**：不要"聪明地"判断哪些产品与用户问题相关，只展示你认为相关的
3. **严禁只展示部分产品**：即使你认为用户的问题只可能涉及某几个产品，也必须展示全部选项
4. **严禁替用户做产品选择**：不要因为觉得"这个问题显然是星空的"就跳过选择步骤直接使用某个产品
5. **产品列表必须完整且按原始顺序展示**：不要重新排序、不要分组、不要隐藏

**错误示例（严禁）：**
> ~用户问"总账怎么初始化"，只展示金蝶AI星空企业版/标准版和金蝶AI星瀚~ ❌
> ~用户问"凭证怎么做"，自动选择金蝶AI星空企业版/标准版~ ❌
> ~只展示3-4个"可能相关"的产品~ ❌
> ~"根据您的问题，以下产品可能相关：..."（然后只列出部分）~ ❌

**正确做法：**
> 无论用户问什么问题，始终展示 `--init` 返回的**全部**产品列表，让用户自己选择。

**必须使用编号列表格式输出，严格按以下格式展示（根据返回的产品列表动态生成）：**

> 请选择您要咨询的产品：
>
> 1. 金蝶AI星瀚
> 2. 金蝶AI套件
> 3. 金蝶AI星辰
> 4. 金蝶AI苍穹
> 5. 金蝶AI星空企业版/标准版
> 6. EAS Cloud
> 7. S-HR Cloud
> 8. 精斗云-云会计
> 9. 精斗云-云进销存
> 10. 账无忧
>
> 请输入编号选择：

**格式要求：**
- **必须使用编号列表**（1. 2. 3. ...），不能用无序列表（- ）、字母列表（a b c）或其他格式
- 编号必须从 1 开始，连续递增
- 每行一个产品，编号后紧跟产品名称
- 用户回复编号后，根据编号匹配对应的 `productId`

**用户选择后：**
- 保存 `selected_product_id` 和 `selected_product_name` 到会话状态
- 后续对话默认使用该产品

**更换产品：**
当用户说"更换产品"、"切换产品"、"我要问其他产品"等时：
1. 重置 `selected_product_id` = null
2. 重新展示产品列表让用户选择

### 步骤3：流式调用接口

执行脚本调用智能问答接口：

```bash
python <skill_dir>/scripts/cosmic_qa.py --question "用户的问题" --product-id <selected_product_id> --session-id <session_id>
```

脚本会自动从本地文件 `~/.kdclub/pat_token.json` 读取 Token，无需额外传参。

**脚本输出格式：**

脚本通过 stdout 输出多行 JSON（JSON Lines），每行一个 JSON 对象：

```
{"type": "start"}
{"type": "think", "content": "...", "step": "intentRecognize"}
{"type": "answer", "content": "...", "step": "generateAnswer"}
{"type": "end", "sessionId": "xxx", "fullAnswer": "完整回答内容", "answerFormat": "markdown", "thinkContent": "...", "searchSources": [...]}
```

**error 类型的 errorCode 说明：**

| errorCode | 含义 | 处理方式 |
|-----------|------|---------|
| `TOKEN_NOT_FOUND` | 未找到 Token | 引导用户获取并提供 Token（步骤1） |
| `UNAUTHORIZED` | Token 无效或过期 | 提示用户重新提供 Token 并用 `--save-token` 保存 |
| （无 errorCode） | 网络/服务端错误 | 提示用户检查网络后重试 |

### ⚠️ 极其重要：输出处理步骤（必须严格按此执行）

**脚本执行完毕后，必须按以下步骤处理输出：**

**第一步：从输出中找到 `type` 为 `end` 的那一行 JSON，提取以下字段：**
- `fullAnswer` —— 完整的回答内容（已格式化，可能是 Markdown 或 HTML）
- `answerFormat` —— 内容格式（`markdown` 或 `html`）
- `sessionId` —— 会话ID
- `searchSources` —— 参考来源列表

**第二步：将 `fullAnswer` 的内容直接展示给用户。** 这是最关键的一步：
- `fullAnswer` 的值就是要展示给用户看的最终内容，**直接输出即可**
- 如果 `answerFormat` 是 `markdown`，直接渲染展示
- 如果 `answerFormat` 是 `html`，保持 HTML 原样输出，**绝对不要**转换为 Markdown
- **不要**在 `fullAnswer` 前面加任何引导语（如"根据查询结果"、"以下是回答"等）
- **不要**在 `fullAnswer` 后面加任何结语（如"希望对您有帮助"等）
- **不要**对 `fullAnswer` 做任何总结、改写、精简、重新组织
- **不要**省略 `fullAnswer` 中的任何图片

**第三步：如果 `searchSources` 非空，在 `fullAnswer` 之后追加「参考来源」版块：**

> **参考来源**
> 1. [文档标题1](文档URL1)
> 2. [文档标题2](文档URL2)
> ...

如果 `searchSources` 为空数组或不存在，则不展示该版块。

**第四步：保存会话状态，供后续追问使用：**
- 将 `sessionId` 保存到会话状态的 `session_id`
- 下次用户追问时传入该 `session_id`

### ⚠️ 展示铁律

1. **`fullAnswer` 必须逐字逐句原封不动展示**：严禁总结、概括、精简、改写、重新组织
2. **所有图片必须完整展示**：`fullAnswer` 中包含多少张图片就必须展示多少张，严禁删减、省略、跳过
3. **不改变格式**：内容是什么格式就展示什么格式，HTML 不转 Markdown
4. **严禁自行判断相关性**：不要判断回答是否与用户问题相关，不要修改用户的提问
5. **一次查询，一次回答**：调用一次接口就展示一次结果，严禁自动重新查询

**保存会话状态：**
- 从 `end` 消息中提取 `sessionId` 保存到会话状态
- 下次用户追问时使用该 sessionId

### 步骤4：多轮对话

**用户继续提问时：**

1. **检查产品**：`selected_product_id` 已设置，直接使用
2. **调用接口**：使用保存的 `session_id` 进行多轮对话（无需再次检查 Token，API 调用本身会处理鉴权失败）

```bash
python <skill_dir>/scripts/cosmic_qa.py --question "用户的追问" --product-id <selected_product_id> --session-id <session_id>
```

**用户要求更换产品时：**

1. 重置 `selected_product_id` = null
2. 重置 `session_id` = null
3. 展示产品列表让用户重新选择（执行步骤2）

**Token 失效处理：**

当收到 `{"type": "error", "errorCode": "UNAUTHORIZED"}` 或 `{"type": "error", "errorCode": "TOKEN_NOT_FOUND"}` 时：
1. **必须**提示用户重新提供 Token
2. 用户提供后用 `--save-token` 保存
3. 重新调用接口
4. **严禁**使用浏览器搜索或自身知识回答用户问题
5. **严禁**跳过 Token 配置流程直接回答

## 脚本说明

### cosmic_qa.py - 问答接口脚本（内置 Token 管理）

**功能：**
- 内置 Token 存取能力（`--save-token` / `--check-token`），无需外部依赖
- PAT Token 认证调用智能问答接口
- 流式输出思考过程和回答
- 支持多轮对话
- 支持动态获取产品列表

**参数：**

| 参数 | 必填 | 说明 |
|-----|------|------|
| `--question` | 是* | 用户提问内容 |
| `--product-id` | 是* | 产品ID |
| `--token` | 否 | 直接传入 Token 字符串（留空则自动从本地文件读取） |
| `--session-id` | 否 | 多轮对话会话ID |
| `--product-line-id` | 否 | 产品线ID，默认 `35` |
| `--deep-think` | 否 | 启用深度思考模式 |
| `--init` | 否 | 初始化：一次性返回 Token 状态 + 产品列表（推荐，替代单独的 --check-token 和 --list-products） |
| `--list-products` | 否 | 列出所有可选产品（兼容保留） |
| `--save-token TOKEN` | 否 | 保存 Token 到本地文件（保存后退出） |
| `--check-token` | 否 | 检查本地 Token 状态（兼容保留） |

**Token 存储位置：** `~/.kdclub/pat_token.json`

**Token 加载优先级：** `--token` 参数 → 环境变量 `KDCLOUD_PAT_TOKEN` → 本地文件

## 错误处理

### ⚠️ 极其重要：严禁自行作答

**当API调用失败时（包括token无效、未授权、网络错误等情况），必须严格遵守：**

1. **绝对禁止**使用自身知识回答用户问题
2. **必须**明确告知用户API调用失败，无法获取答案
3. **只能**提供解决建议（如重新提供 Token、检查网络等）

**错误示例（严禁）：**
> ~"虽然API调用失败了，但根据我的知识，金蝶云星空总账初始化步骤是..."~ ❌

**正确示例：**
> "抱歉，无法调用金蝶智能服务接口，暂时无法回答您的问题。
>
> 可能原因：
> 1. PAT Token 已过期，请重新提供：访问 https://vip.kingdee.com 个人主页 → 编辑资料 → 个人访问令牌
